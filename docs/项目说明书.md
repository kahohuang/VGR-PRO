# VGR PRO AI 助手项目说明书

## 1. 项目定位

这个工具服务于 VGR PRO 在 TikTok 美区的达人运营工作，核心目的是把“达人建联表”里的零散状态整理成可执行的跟进清单，帮助你更快判断：

- 现在该优先跟进谁
- 这个达人处于哪个合作阶段
- 是否需要继续推动样品申请或寄样
- 是否已经发布视频
- 是否已经拿到 `ad code` 或账号授权
- 当前最合适发送什么英文消息

它本质上不是一个通用 CRM，而是一个围绕你当前业务流程定制的轻量运营助手。

## 2. 解决的问题

当前达人运营链路长、状态多，而且很多字段带有经验判断。如果完全靠手动浏览 Excel，会出现几个问题：

- 同一个达人当前阶段不容易快速判断
- 老达人和新达人容易混在一起处理
- 视频发布后，授权跟进容易漏掉
- 日常催样、催发布、催授权的话术切换成本高
- 每天优先事项不够清晰

这个工具通过规则化分析，把这些判断压缩成统一的结果字段和本地可视化页面。

## 3. 当前能力范围

当前版本主要覆盖 4 类能力：

1. 数据读取
读取达人建联 Excel 表中的核心字段。

2. 阶段判断
根据合作状态、视频数、WhatsApp 状态、父亲节标签、授权状态等信息，判断达人当前阶段。

3. 消息建议
根据当前阶段和中文意图，自动生成可直接发送给达人的英文消息建议。

4. 本地输出
生成：

- 每日 CSV 跟进清单
- Markdown 跟进摘要
- HTML Dashboard 本地页面

## 4. 适用场景

这个工具最适合以下场景：

- 每天开始工作前快速看今日优先跟进达人
- 区分老达人复投和新达人首次合作
- 找出已经发布视频但还没给授权的人
- 针对“待申请样品”“待发布”“待授权”等阶段快速复制英文话术
- 在不动原始业务数据的前提下，先做本地分析和页面查看

## 5. 技术方案

当前采用 Python + 原生 HTML/JS 的轻量方案，而不是 React。

原因是：

- 你的核心诉求是本地快速运行，不是多人在线系统
- 主要复杂度在业务规则，不在前端框架
- 现阶段保持依赖轻、启动快，比引入前端工程化更重要

这次重构后，代码已经按职责拆分，后续如果 Dashboard 交互继续复杂化，再考虑前后端分离会更合适。

## 6. 当前仓库边界

Git 仓库只管理：

- Python 代码
- 测试文件
- README
- AI 生成的设计、计划、说明文档

不纳入 Git 的内容包括：

- 业务 Excel 数据
- 达人话术原文件
- 运行生成的输出文件
- 缓存文件

这样可以保证仓库干净，也避免把你的私人项目资料误提交。

## 7. 后续建议

后续迭代优先级建议如下：

1. 先稳定规则
继续把阶段识别、授权识别、优先级判断打磨准确。

2. 再补测试
围绕关键阶段多补一些测试样例，减少以后改规则时的回归风险。

3. 最后再增强页面
如果你后面真的要在 Dashboard 里做更多编辑、联动筛选、状态回写，再考虑是否单独升级前端方案。

## 8. 结论

这个项目的价值不在“做成一个很重的系统”，而在于把你每天重复判断和跟进的动作标准化、结构化，并且尽量保持足够轻、足够稳、足够好改。
